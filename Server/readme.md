# Socket IM实践

## 简介

这是我大一课外作业完成的轻量级 IM（即时通讯）系统，包含单线程 + 多路 IO 复用的 Reactor 架构服务端，以及配套客户端。

- 开发时间：4 月完成核心功能，7 月首次提交 Git(此前未养成使用版本控制的习惯)
- 核心特点：开发时未接触过「Reactor 架构」概念，仅通过对 “多路 IO 复用 + 单线程处理多连接” 的需求推理，自主实现了 Reactor 模式的核心逻辑；
- 项目不足：代码存在内存泄漏、硬编码字符串、指令解析鲁棒性不足等问题，体现了大一阶段的技术稚嫩，但完整跑通了 IM 的核心流程。
  


## 简单说明


### 整体逻辑
```
主线程 Poll 监听所有 Socket FD → 检测到 IO 事件 → 
  ├─ 新连接：创建客户端 FD 并加入监听集合
  ├─ 读事件：读取客户端指令，加入任务队列
  └─ 任务队列处理完成 → 结果写入对应客户端 FD 的专属缓冲区 → 主线程统一将缓冲区数据回写给客户端
```
### 命令处理
实话说，当时知道有json。但并没想到json可以用于命令处理。
命令处理是基于简单的字符串分割

|前缀|内容1|内容2|用途|
|--|--|--|--|
|	login |-- 用户名 |   -- 密码|	用户登录（校验密码 / 登录状态）|
|	logot |-- 用户名 |   -- 空	|用户登出（保存用户数据）|
|	regst |-- 用户名 |   -- 密码|	用户注册（持久化用户信息）|
|	showm |-- 用户名 |   -- 空	|查看当前用户的消息列表|
|	sendm |-- 用户名 |   -- 消息|	给指定用户发送消息|
|	findu |-- 用户名 |   -- 空	|查找用户是否存在|
|	endse |--root    |-- 密码	|管理员关闭服务端|
